## 1. 简介

tinydb 是一个从零构建的 Python 嵌入式关系型数据库，用于教学和学习数据库核心原理。它支持标准 SQL 语法、ACID 事务、并发控制、B-tree 索引、多表 JOIN 查询，并提供交互式 CLI/REPL 界面。

**版本:** v0.2.1

**核心特性：**

- 纯 SQL 字符串接口 `db.execute("SELECT ...")`
- PostgreSQL 风格 SQL 方言
- 多表 JOIN（7 种类型 × 3 种算法）
- 代价优化自动选择 JOIN 算法
- ACID 事务（Shadow Paging + MVCC）
- 并发控制（S/X 锁 + 死锁检测）
- 4 种隔离级别
- B-tree 索引加速
- CLI/REPL（语法高亮、自动补全、执行计划可视化）
- 单文件持久化

## 2. 安装与启动

### 2.1 环境要求

- Python >= 3.10
- 可选依赖: `pygments>=2.10`（用于 SQL 语法高亮）

### 2.2 安装

```bash
# 从源码安装
cd tinydb_opencode
pip install -e .

# 或安装可选依赖
pip install -e ".[pygments]"
```

### 2.3 启动方式

```bash
# 方式一：tinydb 命令（默认数据库文件: tinydb.db）
tinydb

# 方式二：指定数据库文件路径
tinydb mydata.db

# 方式三：Python 模块方式
python3 -m tinydb.cli mydata.db
```

启动后进入交互式 REPL 界面：

```
tinydb> 
```

退出 REPL：

```
tinydb> .exit
```

或按 `Ctrl+D`（EOF）/ `Ctrl+C`（KeyboardInterrupt）。

## 3. REPL 交互界面

### 3.1 基本操作

REPL 以 `tinydb>` 为提示符，支持多行 SQL 输入。SQL 语句以分号 `;` 结束：

```
tinydb> SELECT * FROM users
..  WHERE age > 25;
```

续行提示符为 `.. `，系统通过括号配对判断语句是否完整。

### 3.2 元命令（Dot Commands）

所有元命令以 `.` 开头：

| 命令 | 说明 | 示例 |
|------|------|------|
| `.exit` / `.quit` | 退出 REPL | `.exit` |
| `.tables` | 列出所有用户表 | `.tables` |
| `.schema <table>` | 显示表结构（列名、类型、约束） | `.schema users` |
| `.explain <sql>` | 显示查询执行计划树 | `.explain SELECT * FROM users` |
| `.import <table> <file>` | 从 CSV/JSON 文件导入数据 | `.import users data.csv` |
| `.dump <table> [file]` | 导出表数据到 CSV/JSON 或屏幕 | `.dump users output.json` |
| `.timing on\|off` | 开关查询计时显示 | `.timing on` |
| `.highlight on\|off` | 开关 SQL 语法高亮 | `.highlight off` |
| `.help` | 显示帮助信息 | `.help` |

### 3.3 语法高亮

默认开启。基于 pygments 库实现，对 SQL 关键字（蓝色）、字符串（绿色）、数字（黄色）、注释（灰色）进行 ANSI 着色。

若未安装 pygments，自动静默降级为无高亮模式。

关闭/开启：

```
tinydb> .highlight off
Highlighting disabled.
tinydb> .highlight on
Highlighting enabled.
```

### 3.4 自动补全

按 `Tab` 键触发上下文感知的自动补全：

- `FROM` / `JOIN` / `INTO` / `UPDATE` 之后 → 补全表名
- `SELECT` 之后或输入 `.` → 补全列名
- 其他位置 → 补全 SQL 关键字

### 3.5 历史记录

SQL 命令历史自动保存至 `~/.tinydb_history`，跨会话持久化。使用上/下箭头键浏览历史。

### 3.6 查询计时

开启后每次查询显示执行耗时：

```
tinydb> .timing on
Timing enabled.
tinydb> SELECT * FROM users;
+----+-------+-----+
| id | name  | age |
+----+-------+-----+
| 1  | Alice | 30  |
+----+-------+-----+
1 rows in set
Time: 0.23 ms
```

## 4. SQL 语法参考

### 4.1 数据类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `INTEGER` | 整数 | `42` |
| `FLOAT` | 浮点数 | `3.14` |
| `TEXT` | 字符串 | `'hello'` |
| `BOOLEAN` | 布尔值 | `TRUE` / `FALSE` |

### 4.2 DDL — 数据定义

#### CREATE TABLE

```sql
CREATE TABLE table_name (
    column1 TYPE [PRIMARY KEY] [NOT NULL] [UNIQUE],
    column2 TYPE [NOT NULL],
    ...
);
```

示例：

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    age INTEGER,
    email TEXT UNIQUE
);
```

支持的约束：
- `PRIMARY KEY` — 主键（隐式 NOT NULL）
- `NOT NULL` — 非空约束
- `UNIQUE` — 唯一约束

#### DROP TABLE

```sql
DROP TABLE table_name;
```

#### CREATE INDEX

```SQL
CREATE INDEX index_name ON table_name (column_name);
```

### 4.3 DML — 数据操作

#### INSERT

```sql
-- 插入单行
INSERT INTO users VALUES (1, 'Alice', 30);

-- 插入单行（指定列）
INSERT INTO users (id, name, age) VALUES (1, 'Alice', 30);

-- 批量插入多行
INSERT INTO users VALUES 
    (1, 'Alice', 30),
    (2, 'Bob', 25),
    (3, 'Charlie', 35);
```

字符串使用单引号包裹。`NULL` 表示空值。

#### SELECT

```sql
SELECT column1, column2, ...
FROM table_name
[WHERE conditions]
[ORDER BY column [ASC|DESC]]
[LIMIT count]
[LIMIT count OFFSET offset];
```

示例：

```sql
-- 查询所有列
SELECT * FROM users;

-- 查询指定列
SELECT name, age FROM users;

-- 条件查询
SELECT * FROM users WHERE age > 25;

-- 排序
SELECT * FROM users ORDER BY age DESC;

-- 分页
SELECT * FROM users LIMIT 10 OFFSET 20;
```

#### UPDATE

```sql
UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE conditions;
```

示例：

```sql
UPDATE users SET age = 31 WHERE id = 1;
UPDATE users SET name = 'Alice Wang', age = 32 WHERE id = 1;
```

> **注意：** UPDATE 必须包含 WHERE 子句，否则会更新全表。

#### DELETE

```sql
DELETE FROM table_name
WHERE conditions;
```

示例：

```sql
DELETE FROM users WHERE id = 1;
DELETE FROM users WHERE age < 18;
```

> **注意：** DELETE 必须包含 WHERE 子句，否则会删除全表数据。

### 4.4 WHERE 条件

支持运算符：

| 运算符 | 说明 |
|--------|------|
| `=` | 等于 |
| `!=` / `<>` | 不等于 |
| `>` / `>=` | 大于 / 大于等于 |
| `<` / `<=` | 小于 / 小于等于 |
| `AND` | 逻辑与 |
| `OR` | 逻辑或 |
| `NOT` | 逻辑非 |
| `IS NULL` | 为空 |
| `IS NOT NULL` | 非空 |

优先级（从高到低）：`NOT` > `AND` > `OR`

示例：

```sql
SELECT * FROM users WHERE age > 25 AND name = 'Alice';
SELECT * FROM users WHERE age < 18 OR age > 65;
SELECT * FROM users WHERE NOT (age < 25);
SELECT * FROM users WHERE email IS NULL;
```

### 4.5 多表 JOIN 查询

#### JOIN 类型

| 类型 | 语法 | 说明 |
|------|------|------|
| INNER JOIN | `A JOIN B ON condition` | 返回两表匹配行 |
| LEFT JOIN | `A LEFT JOIN B ON condition` | 返回左表全部 + 右表匹配行（不匹配补 NULL） |
| RIGHT JOIN | `A RIGHT JOIN B ON condition` | 返回右表全部 + 左表匹配行（不匹配补 NULL） |
| FULL OUTER JOIN | `A FULL OUTER JOIN B ON condition` | 返回两表全部行（不匹配补 NULL） |
| CROSS JOIN | `A CROSS JOIN B` | 笛卡尔积 |
| NATURAL JOIN | `A NATURAL JOIN B` | 自动按同名列等值连接 |

#### JOIN 示例

```sql
-- INNER JOIN
SELECT u.name, o.amount 
FROM users u 
JOIN orders o ON u.id = o.user_id;

-- LEFT JOIN
SELECT u.name, o.amount 
FROM users u 
LEFT JOIN orders o ON u.id = o.user_id;

-- 多表 JOIN
SELECT u.name, o.amount, p.title
FROM users u
JOIN orders o ON u.id = o.user_id
JOIN products p ON o.product_id = p.id;

-- 自连接（SELF JOIN）
SELECT a.name, b.name 
FROM employees a 
JOIN employees b ON a.manager_id = b.id;
```

#### JOIN 算法

系统根据代价模型自动选择最优算法：

| 算法 | 适用场景 |
|------|----------|
| Nested Loop Join | 小表连接 |
| Hash Join | 大表等值连接 |
| Sort-Merge Join | 已排序数据连接 |

### 4.6 聚合函数与 GROUP BY

#### 聚合函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `COUNT(*)` | 计数 | `SELECT COUNT(*) FROM users` |
| `COUNT(column)` | 非空计数 | `SELECT COUNT(email) FROM users` |
| `SUM(column)` | 求和 | `SELECT SUM(amount) FROM orders` |
| `AVG(column)` | 平均值 | `SELECT AVG(age) FROM users` |

#### GROUP BY

```sql
SELECT department, COUNT(*), AVG(salary)
FROM employees
GROUP BY department;
```

### 4.7 SHOW TABLES

列出当前数据库中所有用户表：

```sql
SHOW TABLES;
```

## 5. 事务管理

### 5.1 基本事务操作

```sql
-- 开启事务
BEGIN;

-- 执行 DML 操作
INSERT INTO users VALUES (1, 'Alice', 30);
UPDATE users SET age = 31 WHERE id = 1;
DELETE FROM users WHERE id = 2;

-- 提交事务（所有修改原子生效）
COMMIT;

-- 或回滚事务（撤销所有修改）
ROLLBACK;
```

### 5.2 事务 ACID 特性

| 特性 | 实现方式 |
|------|----------|
| 原子性 (Atomicity) | Shadow Paging — 写时复制，提交时原子切换根指针 |
| 一致性 (Consistency) | 约束检查 + 事务回滚保证数据一致 |
| 隔离性 (Isolation) | S/X 锁 + MVCC 多版本并发控制 |
| 持久性 (Durability) | `fsync` 确保数据落盘 |

### 5.3 隔离级别

| 级别 | 说明 |
|------|------|
| `READ UNCOMMITTED` | 最低级别，可能读到未提交数据 |
| `READ COMMITTED` | 只读已提交数据 |
| `REPEATABLE READ` | 默认级别，事务内读取一致性快照 |
| `SERIALIZABLE` | 最高级别，事务串行执行 |

默认隔离级别为 `REPEATABLE READ`。

### 5.4 并发控制

- **Shared Lock (S 锁)**: 读操作自动获取，多个事务可同时持有
- **Exclusive Lock (X 锁)**: 写操作自动获取，独占访问
- **锁超时**: 等待超过 5 秒自动超时并回滚事务
- **S→X 升级**: 持有 S 锁的事务可升级为 X 锁（仅当无其他持有者时）
- **死锁检测**: 等待图 DFS 环路检测 + 超时双重保障
- **牺牲者选择**: 最年轻事务优先中止

### 5.5 错误自动回滚

事务中 DML 执行出错时，系统自动回滚当前事务，不污染数据：

```sql
tinydb> BEGIN;
tinydb> INSERT INTO users VALUES (1, 'Alice', 30);
tinydb> INSERT INTO nonexistent VALUES (1);  -- 错误
Error: Table 'nonexistent' not found
-- 事务已自动回滚
```

## 6. 索引管理

### 6.1 创建索引

```sql
CREATE INDEX idx_name ON users (name);
CREATE INDEX idx_age ON users (age);
```

### 6.2 索引使用

索引由查询优化器自动选择，无需手动指定：

```sql
-- 等值查询自动使用索引
SELECT * FROM users WHERE id = 42;

-- 范围查询自动使用索引
SELECT * FROM users WHERE age > 25 AND age < 35;
```

### 6.3 查看执行计划

使用 `.explain` 查看查询是否命中索引：

```
tinydb> .explain SELECT * FROM users WHERE id = 1;
Project [columns=[id, name, age]]
  Filter [condition=id = 1]
    Scan [table=users]
```

### 6.4 索引维护

- INSERT/UPDATE/DELETE 操作自动维护索引一致性
- B-tree 节点分裂时保持平衡
- 删除采用懒删除策略（不合并节点）

## 8. Python API 使用

### 8.1 基本使用

```python
from tinydb import Database

# 打开数据库（自动创建）
db = Database("mydata.db")

# 建表
db.execute("CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT, age INTEGER)")

# 插入数据
db.execute("INSERT INTO users VALUES (1, 'Alice', 30)")
db.execute("INSERT INTO users VALUES (2, 'Bob', 25)")

# 查询
result = db.execute("SELECT * FROM users WHERE age > 25")
print(result.columns)   # ['id', 'name', 'age']
print(result.rows)      # [[1, 'Alice', 30]]
print(result.row_count) # 1

# 关闭
db.close()
```

### 8.2 上下文管理器

```python
with Database("mydata.db") as db:
    db.execute("INSERT INTO users VALUES (1, 'Alice', 30)")
    result = db.execute("SELECT * FROM users")
    for row in result.rows:
        print(row)
# 自动关闭
```

### 8.3 事务控制

```python
with Database("mydata.db") as db:
    db.execute("BEGIN")
    db.execute("INSERT INTO users VALUES (1, 'Alice', 30)")
    db.execute("INSERT INTO users VALUES (2, 'Bob', 25)")
    db.execute("COMMIT")
    
    # 或回滚
    db.execute("BEGIN")
    db.execute("INSERT INTO users VALUES (3, 'Charlie', 35)")
    db.execute("ROLLBACK")
```

### 8.4 QueryResult 对象

| 属性 | 类型 | 说明 |
|------|------|------|
| `columns` | `list[str]` | 列名列表 |
| `rows` | `list[list]` | 数据行列表 |
| `row_count` | `int` | 结果行数 |

### 8.5 获取表结构信息

```python
with Database("mydata.db") as db:
    info = db.get_table_info("users")
    # [{'name': 'id', 'type': 'INTEGER', 'nullable': False, 'primary_key': True},
    #  {'name': 'name', 'type': 'TEXT', 'nullable': True, 'primary_key': False},
    #  {'name': 'age', 'type': 'INTEGER', 'nullable': True, 'primary_key': False}]
```

## 9. 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                      SQL Engine                          │
│  Lexer → Parser → Planner → Executor                    │
│  (Scan/Filter/Project/Aggregate/Sort/Limit/Join)         │
│  JOIN Algorithms: NestedLoop / Hash / SortMerge          │
└──────────────────────┬──────────────────────────────────┘
                       │ Table API
┌──────────────────────┴──────────────────────────────────┐
│                   Storage Engine                         │
│  TypeSystem → RowFormat → Page → FileManager            │
│  BufferPool (LRU + pin/unpin + Page Latch)              │
│  Catalog (tinydb_master)                                │
│  B-tree Index + IndexManager                            │
│  Transaction (Shadow Paging)                            │
├─────────────────────────────────────────────────────────┤
│                  Concurrency Layer                        │
│  LockManager (S/X locks, compatibility matrix, timeout) │
│  MVCCManager (version chain, snapshot, GC)              │
│  DeadlockDetector (wait-for graph, cycle detection)     │
│  Isolation Levels (4 levels, default REPEATABLE READ)   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                     CLI/REPL                             │
│  SQLHighlighter (pygments) / SQLCompleter (readline)    │
│  Commands: .explain / .import / .dump / .timing /       │
│            .highlight                                    │
│  历史持久化 (~/.tinydb_history)                          │
└─────────────────────────────────────────────────────────┘
```

### 数据存储

- 数据以单 `.db` 文件格式持久化存储
- 页大小固定 4KB，与 OS 内存页对齐
- 文件头包含 magic bytes `TINYDB\0` 与 checksum 校验
- 重启后数据和索引结构完整恢复

## 10. 常见问题

### Q: 如何指定数据库文件路径？

```bash
tinydb path/to/mydata.db
```

或在 Python 中：

```python
db = Database("path/to/mydata.db")
```

### Q: 如何查看当前数据库有哪些表？

```sql
SHOW TABLES;
```

或使用元命令：

```
tinydb> .tables
```

### Q: 如何查看表结构？

```
tinydb> .schema users
```

或 Python API：

```python
db.get_table_info("users")
```

### Q: 如何查看查询执行计划？

```
tinydb> .explain SELECT * FROM users WHERE id = 1;
```

### Q: 字符串中使用单引号怎么办？

当前版本不支持转义单引号。建议避免在数据中使用单引号，或在应用层预处理。

### Q: 事务中出错怎么办？

系统会自动回滚当前事务。你也可以手动执行 `ROLLBACK`。

### Q: 如何备份数据库？

直接复制 `.db` 文件即可。或使用导出功能：

```
tinydb> .dump tablename backup.csv
```

### Q: 支持哪些 Python 版本？

Python >= 3.10。

### Q: 并发写入安全吗？

是的。系统通过页级 S/X 锁保证多线程读写安全。读不阻塞读（S-S 兼容），写操作独占（X 锁）。死锁由等待图 + 超时双重检测并自动恢复。
