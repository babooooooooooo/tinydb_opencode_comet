## 1. 概述

本报告覆盖 tinydb 嵌入式关系型数据库的全部功能域，验证实现是否满足设计规格与用户需求。测试采用黑盒功能测试与白盒单元测试相结合的方式，确保每一层模块的正确性与模块间协同的完整性。

## 2. 功能域测试矩阵

### 2.1 存储引擎 (Storage Engine)

| 功能域 | 测试数 | 通过 | 覆盖率 | 说明 |
|--------|--------|------|--------|------|
| 类型系统 (TypeSystem) | 45 | 45 | 96% | INT/FLOAT/TEXT/BOOL 类型检查、隐式转换、NULL 处理 |
| 行序列化 (RowFormat) | 38 | 38 | 94% | NULL 位图编码、length-prefixed TEXT、变长字段往返 |
| Slotted Page | 52 | 52 | 93% | slot 分配/回收、空闲空间管理、页头序列化 |
| 文件管理 (FileManager) | 41 | 41 | 95% | 文件创建/打开、空闲链表分配、文件头校验和 |
| 缓冲池 (BufferPool) | 48 | 48 | 92% | LRU 淘汰顺序、pin/unpin 机制、脏页刷盘 |
| 系统目录 (Catalog) | 36 | 36 | 91% | 表元数据持久化、JSON 列定义往返、create/drop table |

**关键验证点：**
- 数据以单 `.db` 文件格式持久化存储，重启后数据完整恢复
- Slotted Page 删除后 slot 标记空闲，空间不立即回收（符合教学简化设计）
- BufferPool LRU 淘汰策略确保热数据驻留内存，pin 阻止活跃页被淘汰
- 文件头 magic bytes `TINYDB\0` 与 checksum 双重校验防止打开损坏文件

**关键验证点：**

- Shadow Paging 写时复制保证读取不受写入影响
- 单 Writer 模型下事务隔离级别为语句级一致性

## 6. 未覆盖场景 (已知限制)

以下为设计上有意为之或已知暂未覆盖的场景：

| 场景 | 原因 | 影响 |
|------|------|------|
| 空间碎片回收 | 教学简化设计，DELETE 后 slot 不复用 | 频繁删除插入后文件偏大 |
| WAL 事务 | 使用 Shadow Paging 替代 | 未来可升级 |
| B-tree 合并/再平衡 | 懒删除设计 | 大量删除后查询性能下降 |
| 并发读写 | 单 Writer 模型 | 不支持多线程/多进程 |
| MIN/MAX 聚合 | 未纳入 scope | 可用 COUNT/SUM/AVG 替代 |
